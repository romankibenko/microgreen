# Telegram-бот

Бот на **aiogram 3** (long polling). Держит клиента в курсе заказов и связывает
телефон с Telegram-аккаунтом, чтобы заказы с лендинга находили своего владельца.

## Что умеет

- `/start` — приветствие и меню.
- 📋 **Каталог** — что сейчас в наличии (тянет из backend `/products`).
- 🧾 **Мои заказы** — статусы заказов на привязанный номер (🆕 принят · 📦 собран · ✅ доставлен · ❌ отменён).
- 📱 **Поделиться телефоном** — привязка номера к чату (`/telegram/link`), чтобы получать уведомления.
- `/id` — показывает `chat_id` (нужно Роме, чтобы вписать `ADMIN_CHAT_ID`).
- 🛒 **Магазин** — кнопка-заглушка под будущий Mini App (показывается только при `SHOP_URL` на HTTPS).

Уведомления о новом заказе шлёт **backend** (через Bot API), а не бот — бот занят только
входящими командами. Токен общий, лежит в `.env`.

## Переменные окружения

| Переменная | Назначение |
|------------|-----------|
| `BOT_TOKEN` | Токен от [@BotFather](https://t.me/BotFather). Без него бот не стартует. |
| `BACKEND_URL` | Адрес API (в Compose — `http://backend:8000`). |
| `SHOP_URL` | URL каталога для кнопки «Магазин» (Mini App требует HTTPS). |

## Как завести бота (BotFather)

1. Напиши [@BotFather](https://t.me/BotFather) → `/newbot`, задай имя и username.
2. Скопируй токен → впиши в корневой `.env` как `BOT_TOKEN=...` (раскомментируй строку).
3. Подними стек: `docker compose up --build`.
4. Открой своего бота, нажми `/start`, затем `/id` → скопируй `chat_id`.
5. Впиши его в `.env` как `ADMIN_CHAT_ID=...` и перезапусти стек — теперь уведомления о заказах прилетают тебе.

## Запуск

Отдельно бот не нужен — он часть Compose-стека (`docker compose up`). Без `BOT_TOKEN`
контейнер логирует предупреждение и завершается (без рестарт-лупа).

# План разработки — «Микрозелень»

Принцип: **итеративно, кейс показуем как можно раньше.** После Этапа 2 уже есть что показать в портфолио, после Этапа 5 — живой проект в проде. Всё после — наращивание галочек.

Каждый этап заканчивается проверяемым критерием (✅). Не переходим дальше, пока критерий не выполнен.

---

## Этап 0 — Скелет и инфраструктура ✅ ГОТОВО
- Монорепо, `git init`, репозиторий на GitHub (для портфолио GitHub обязателен).
- `backend/`, `landing/`, `admin/`, `bot/` — пустые скелеты.
- `docker-compose.yml`: postgres + backend поднимаются.
- `.env.example`, `.gitignore`, `README.md` (краткий, для GitHub-витрины).
- ✅ `docker compose up` поднимает PG + FastAPI с `/health`, отвечающим 200.

## Этап 1 — Backend-ядро ✅ ГОТОВО
- Модели: `products` (товары), `orders` + `order_items` (заказы), `plantings` (журнал посадок: дата посева → авторасчёт даты готовности).
- Alembic-миграции.
- API: каталог товаров (GET), создание заказа (POST, гостевое — телефон + позиции), список заказов.
- ✅ Через Swagger можно создать товар, оформить заказ, получить список заказов. Миграции применяются с нуля.

## Этап 2 — Лендинг (первая показуемая точка!) ✅ ГОТОВО
- Vue 3 + Vuetify: каталог из API, корзина (Pinia), форма заказа → POST в API.
- Адаптив, нормальные фото микрозелени.
- ✅ С телефона можно открыть сайт, добавить в корзину, оформить заказ → он появляется в БД. **Это уже скриншотится в портфолио.**

## Этап 3 — Telegram-бот ✅
- aiogram 3: при новом заказе → уведомление Роме (новый заказ, состав, телефон) + подтверждение клиенту (если он в боте).
- Команды: каталог, мои заказы, статус.
- Заложить кнопку-заглушку «🛒 Магазин» под будущий Mini App (см. «После MVP») — чтобы потом просто подвесить URL.
- ✅ Оформил заказ на лендинге → прилетела нотификация в Telegram.

## Этап 4 — Админка ◄ ТЕКУЩИЙ
- Vue 3, закрытый роут, JWT-логин.
- Список заказов со сменой статуса (новый → собран → доставлен).
- Журнал посадок с датами готовности (что и когда срезать).
- ✅ Логинюсь, вижу заказы, меняю статус, веду посадки. Без токена — редирект на логин.

## Этап 5 — Деплой (превращение в ЖИВОЙ кейс)
- Docker Compose на VDS, Nginx reverse proxy, домен, HTTPS (Let's Encrypt).
- UptimeRobot + healthcheck + алерт в Telegram.
- ✅ Сайт открывается по домену с замочком. Полный путь заказ→бот→админка работает на проде. **Ссылка идёт в портфолио.**

## После MVP — портфолио-фичи (по одной, осознанно)
- **Telegram Mini App** — тот же Vue-фронт-каталог, открытый внутри бота по кнопке «🛒 Магазин». Не дублирует лендинг: один фронт обслуживает и браузер, и Mini App, заказы падают в то же API. Авторизация через Telegram (без ввода телефона), позже — нативная оплата.
  ⚠️ **Зависит от Этапа 5:** Mini App открывается только по HTTPS на публичном домене — делать ПОСЛЕ деплоя (локально Telegram webview не откроет, нужен ngrok-туннель = лишний геморрой). Это редкий горячий навык → жирный плюс в портфолио.
- **ЮKassa**: онлайн-оплата + webhook с проверкой подписи. Ценно как демо платёжной интеграции.
- **LLM-консультант** в боте (OpenRouter): «спроси про пользу/рецепты». Галочка «GPT/LLM».
- **Подписка** «Зелёная коробка» с автоплатежом.
- **Аналитика** на дашборде (Chart.js): выручка, топ-товары, воронка.

## Как резать на работы в портфолио
Один проект = до 5 кейсов на Kwork/FL:
1. Лендинг с корзиной и заявками → бот
2. Telegram-бот (уведомления, каталог, позже LLM)
3. Админ-панель с аналитикой
4. REST API + интеграция ЮKassa
5. Telegram Mini App (магазин внутри Telegram) — редкий навык, отдельный кейс
